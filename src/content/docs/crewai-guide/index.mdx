- **v1.14.4 (May 1)** — Patch release; stability improvements.

## What's new in v1.14.3 (April 2026)

- **New reference pages** — Agent, Task, Memory, Knowledge, Tools, LLM, CLI — each verified against installed source.
- **Checkpoint forking with lineage tracking** — branch execution from any saved checkpoint.
- **`from_checkpoint` on `Crew.kickoff` / `Flow.kickoff`** — resume a crew or flow with `from_checkpoint=CheckpointConfig(restore_from=...)`.
- **Checkpoint CLI** — `crewai checkpoint list|info|resume|diff|prune`.
- **Template management CLI** — `crewai template list|add`.
- **Enriched token tracking** — `reasoning_tokens`, `cache_creation_tokens` on `CrewOutput.token_usage`.
- **Scoped streaming handlers** — per-run isolation across concurrent crews.

<Aside type="caution" title="Breaking changes since v1.5.0">
  - `CodeInterpreterTool` removed from the public API — use E2B / Modal / a custom `BaseTool`.
  - `CrewStructuredTool` moved to internal API — external code should use `BaseTool`.
  - `stop=` parameter stripped for GPT-5 / o-series models by the LLM router.
  - `Agent.reasoning` deprecated → use `planning` + `PlanningConfig`.
  - `Agent.multimodal` deprecated → use `Task.input_files={"name": path}`.
  - `from crewai.amp import ...` **is not a valid import** in the stable package. AMP is accessed through the CLI (`crewai login`, `crewai deploy ...`). The AMP Suite section of the comprehensive guide includes pre-existing fabricated imports and is flagged with an errata note.
  - v1.10.0 yanked — use 1.10.1+.
</Aside>

## Install, minimal example

```python
from crewai import Agent, Task, Crew, Process, LLM

llm = LLM(model="openai/gpt-4o-mini")

analyst = Agent(
    role="Research Analyst",
    goal="Analyse topics thoroughly",
    backstory="Expert analyst with 10 years of experience",
    llm=llm,
)

task = Task(
    description="Research AI advancements in 2026",
    expected_output="Concise analysis of key developments",
    agent=analyst,
)

crew = Crew(agents=[analyst], tasks=[task], process=Process.sequential)
result = crew.kickoff()
print(result)
```
