### 2.2 LlamaIndex TypeScript — @llamaindex/workflow 1.1.24 → 1.1.25

**Install command:** `npm install @llamaindex/workflow@latest` in `.routine-envs/check-0504-node/`  
**Resolved version:** 1.1.25  
**Install date:** 2026-05-04

**Changelog (from `node_modules/@llamaindex/workflow/CHANGELOG.md`):**

```
## 1.1.25 — Patch Changes
- 5977b1f: Add deprecation message
- Updated dependencies [5977b1f]
  - @llamaindex/core@0.6.23
  - @llamaindex/env@0.1.31
```

**Source findings:**

- `@llamaindex/core` 0.6.23 (a peer dependency of `@llamaindex/workflow`) now carries `"deprecated": "This package is deprecated and no longer maintained."` in its `package.json` (path: `.routine-envs/check-0504-node/node_modules/@llamaindex/core/package.json`). The npm registry does not emit a deprecation warning on install; the field is metadata only. `@llamaindex/workflow` 1.1.25 still lists `@llamaindex/core@0.6.23` as a peer dependency, so users must still install it alongside `@llamaindex/workflow`.

- `runWorkflow()` and `runAndCollect()` are now marked `@deprecated` in `node_modules/@llamaindex/workflow-core/stream/run.js` with the instruction to use `stream.until().toArray()` instead. This is surfaced via the `run()` helper: `run(workflow, event).until(stopEvent).toArray()`.

- `handle()` on a workflow object mutates state in-place and returns `void` (confirmed in both 1.1.24 and 1.1.25 source: `.routine-envs/check-0504-node/node_modules/@llamaindex/workflow-core/dist/index.js`, function `createWorkflow()`). Method chaining is not supported.

- Handler signature confirmed by source (`context.handler(workflowContext, ...context.inputs)`): the first argument is the workflow context (`{ stream, signal, sendEvent }`); subsequent arguments are the matched input events in order.

**Changes applied:**

1. **Version bump** — `index.mdx` card: `1.1.24` → `1.1.25`, month `April` → `May`.
2. **Minimal example fixed** — three bugs corrected:
   - Removed unsupported chaining (`createWorkflow().handle(...)` → separate `workflow.handle(...)` call).
   - Fixed handler signature from `async ({ data })` to `async (ctx, start)`, with `start.data.input` used correctly.
   - Replaced deprecated `runWorkflow(workflow, event)` (missing third argument, now deprecated) with `run(workflow, event).until(stopEvent).toArray()`. Import changed from `runWorkflow` to `run`.
3. **"What's new" section** — replaced "1.1.24 (April 2026)" with "1.1.25 (May 2026)" documenting the deprecations and API clarifications.
4. **Revision history entry** added.
